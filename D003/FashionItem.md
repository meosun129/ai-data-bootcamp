# 옷장패션 데이터 정제 보고서

## 1. 데이터 개요
- 행/열: 1,500 × 8
- 주요 컬럼: order_id, customer_age, category, channel, price, quantity, amount, return_amount

## 2. 진단 결과
- 결측: amount 3.40%, price 0.33%
- 이상치(IQR, k=1.5):
  - customer_age: 999·0 입력 오류 5건 포함 총 18건 (나머지는 분포 꼬리)
  - quantity: 200(단일 소비자 대량 구매 추정) 1건
  - amount: 상한 253,200원 초과 총 145건 — 이 중 1건은 50,000,000원(도매 의심 극단치)이고, 나머지 144건은 고가×다수량 조합(259,800원 95건, 389,700원 49건)으로 정상 거래에 가까움. 일괄 제거는 부적절, 별도 검토 필요
  - return_amount: 값의 92%↑가 0이라 Q1=Q3=0, IQR=0 → 상한이 0이 되어 반품 발생 건(122건)이 전부 "이상치"로 잡힘. IQR 방법 자체가 부적합한 케이스로 판단, 적용 제외
- 의심되는 결측 유형: amount는 app 채널에 쏠림 → MAR

## 3. 처리 결정과 근거
1. 결측
- amount : amount는 app 채널에 쏠려 app에서 오류가 났을 MAR임을 알았으나, 
amount=price*quantity이므로 amount가 nan인 값에 대하여 해당 식을 이용하여 nan인 amount 값을 복원한다.
- price : price 또한 amount=price*quantity을 이용하여 구하고, [15900, 29900, 49900, 79900, 129900] 중에 값이 있는지 확인한 후 복원한다.

2. 이상치
- customer_age: 물리적 불가능 값 → NaN → 중앙값 대체 vs 평균값 대체
- quantity: 200(단일 소비자 대량 구매 추정) 1건에 대한 데이터는 따로 관리
- amount: 50,000,000원(1건)은 정상 조합 최댓값(389,700원)을 초과하는 명백한 입력 오류로 판단 → price*quantity로 정정하여 제거. 나머지 상한(253,200원) 초과 건은 고가×다수량 정상 조합으로 판단, 유지.
- return_amount: 값의 92%↑가 0이라 Q1=Q3=0, IQR=0 → 상한이 0이 되어 반품 발생 건(122건)이 전부 "이상치"로 잡힘. IQR 방법 자체가 부적합한 케이스로 판단, 적용 제외.
